## ūüďĆ What is MVSR Connect?

MVSR Connect is a **Reddit-style campus forum** built exclusively for students of MVSR Engineering College. It's a single platform where you can share posts, join clubs, vote on content, have threaded discussions, report inappropriate material, and find lost items ‚ÄĒ all locked behind Google OAuth so only real `@mvsrec.edu.in` accounts can get in.

Think of it as a private, AI-moderated version of a discussion forum, but built specifically for your campus.

## ‚ú® Features

### Core
- ūüĒź **Google OAuth Login** ‚ÄĒ Restricted to `@mvsrec.edu.in` emails only. No external signups.
- ūüďį **Feed** ‚ÄĒ Create posts with text, images, or videos. Sort by **Hot**, **Top**, or **Recent**.
- ūüó≥ÔłŹ **Reddit-style Voting** ‚ÄĒ Upvote and downvote posts. Hot ranking uses a time-decay algorithm.
- ūüí¨ **Threaded Comments** ‚ÄĒ Nested replies with full toxicity filtering before publishing.
- ūüŹ∑ÔłŹ **Tags** ‚ÄĒ Tag posts and filter the feed by tag (Announcement, Event, Question, etc.).
- ūüĒć **Global Search** ‚ÄĒ Search across posts, users, clubs, and tags simultaneously.
- ūüďé **Media Uploads** ‚ÄĒ Images and videos via Cloudinary. Images are auto-compressed client-side before upload.

### Clubs
- ūüé≠ Join clubs (Drama, Music, Photo/Videography, Book, Dance, and more)
- ūüď¨ Submit join requests; moderators approve or reject
- ūüďč Post within clubs ‚ÄĒ only members can post inside a club
- ūüõ°ÔłŹ Apply to become a club moderator via the appeal system

### Lost & Found
- ūüĒé **Report Lost Items** ‚ÄĒ Post what you lost with location, date, category, and a photo
- ūüéČ **Report Found Items** ‚ÄĒ Post what you found so the owner can claim it
- ūüôč **Claim System** ‚ÄĒ Finders respond to lost posts; owners respond to found posts with proof of ownership
- ūüĒí **Private Responses** ‚ÄĒ Only the original poster can see who responded, protecting user privacy
- ‚úď **Resolve Cases** ‚ÄĒ Mark your post resolved once the item is returned
- ūüĒć **Search** ‚ÄĒ Search across item names, descriptions, locations, and categories
- ūü§Ė **Moderation** ‚ÄĒ All text and images run through the same AI toxicity and safety pipeline

### Events
- ūüďÖ **Create Events** ‚ÄĒ Moderators/admins create events with title, venue, date, capacity, and optional banner
- ūüíį **Free & Paid Events** ‚ÄĒ Free events issue QR tickets instantly; paid events use a UPI payment + UTR verification flow
- ūüéüÔłŹ **QR Tickets** ‚ÄĒ Confirmed enrollments get a unique QR code ticket generated server-side
- ūüď∑ **Live QR Scanner** ‚ÄĒ Camera-based scanner page for moderators to scan and check in attendees at the venue
- ūüĒí **UPI Privacy** ‚ÄĒ UPI IDs are stored server-side only and never exposed in any API response
- ūüďä **Enrollment Manager** ‚ÄĒ Organizers can view all enrollments, approve/reject payment submissions, and track check-ins
- ūüŹ∑ÔłŹ **Club-linked Events** ‚ÄĒ Events can optionally be tied to a specific club

### Moderation & Admin
- ūü§Ė **AI Toxicity Detection** ‚ÄĒ Every comment and post is checked by an external Flask microservice before going live
- ūüĖľÔłŹ **Image & Video Safety** ‚ÄĒ Unsafe media is rejected and automatically deleted from Cloudinary
- ūüďĘ **Report System** ‚ÄĒ Users can report posts; admins resolve them
- ūüßĎ‚Äć‚öĖÔłŹ **Mod Panel** ‚ÄĒ Moderators manage their club's members, posts, comments, and join requests
- ūüĎĎ **Admin Panel** ‚ÄĒ Admins approve/reject moderator appeals and resolve all reports

### Notifications
- **In-App Notifications** ‚ÄĒ Real-time notification bell with unread badge and dropdown history
- ūüĆź **Web Push Notifications** ‚ÄĒ Native device notifications (even when tab is closed)
- ‚ö° **Event-Based Alerts**:
    - New comments on your posts
    - Replies to your comments
    - Club join request approvals/rejections
    - Moderator appeal updates
    - Lost & Found responses
    - Event enrollment confirmations
    - New posts in your clubs
- ūüď¶ **Persistent History** ‚ÄĒ All notifications stored in DB and visible in dropdown
- ūüĒē **Smart Permission Handling** ‚ÄĒ Shows ‚ÄúEnable notifications‚ÄĚ prompt only when needed

### Personal
- ūüďä **Dashboard** ‚ÄĒ View your posts, comments, clubs, upvotes received, and liked posts
- ‚úŹÔłŹ **Profile Editor** ‚ÄĒ Update your display name, bio, and avatar

## ūüöÄ Getting Started

### Prerequisites

- Java 17
- PostgreSQL (running locally or remotely)
- Google Cloud project with OAuth2 credentials
- Cloudinary account
- *(Optional)* Flask moderation microservice running on port `5001`

### Environment Variables

Set these in your deployment platform (e.g., Render):

```bash
# Database
DB_URL=your_postgresql_connection_url
DB_USERNAME=your_db_username
DB_PASSWORD=your_db_password

# Google OAuth2
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

# Cloudinary
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret

# Moderation (Flask service)
MODERATION_URL=your_moderation_service_url
MODERATION_ENABLED=true

# HuggingFace (used by moderation service if applicable)
HUGGINGFACE_API_KEY=your_huggingface_api_key

# Email (Gmail SMTP)
MAIL_USERNAME=your_email@gmail.com
MAIL_PASSWORD=your_gmail_app_password

# Web Push Notifications (VAPID)
VAPID_PUBLIC_KEY=your_vapid_public_key
VAPID_PRIVATE_KEY=your_vapid_private_key
VAPID_SUBJECT=mailto:your_email@gmail.com
```
```md id="env-notes"
> ‚ö†ÔłŹ Notes:
> - `MAIL_PASSWORD` must be a Gmail **App Password**, not your actual password
> - `VAPID_SUBJECT` is just an identifier (use any valid email with `mailto:` prefix)
> - All variables must be set in your hosting platform (e.g., Render) ‚ÄĒ do not hardcode secrets
```
### Run Locally
```bash
./mvnw spring-boot:run
```

## ūü§Ė Content Moderation

All user-generated content is automatically screened before it goes live:

- **Text** ‚ÄĒ Posts, comments, and Lost & Found descriptions are checked for toxicity via the Flask microservice at `/check_text`.
- **Images** ‚ÄĒ Image URLs are sent to `/check_image`. If flagged, the image is deleted from Cloudinary and the post is rejected.
- **Videos** ‚ÄĒ Same flow via `/check_video`.

To disable in development:
```properties
MODERATION_ENABLED=false
```

If the Flask service is unavailable, moderation **fails open** (content is allowed through) so the app stays functional.

## ūüß† Hot Ranking Algorithm

Posts in the **Hot** feed are ranked using a Reddit-inspired time-decay formula:
```sql
SELECT p.*
FROM posts p
LEFT JOIN votes v ON p.id = v.post_id
GROUP BY p.id
ORDER BY
  COALESCE(SUM(v.value), 0) /
  POWER(EXTRACT(EPOCH FROM (NOW() - p.created_at)) / 3600 + 2, 1.5)
DESC
```

Where the numerator is the net vote score and the denominator grows over time. The `+2` prevents brand-new posts from dividing by zero, and the `1.5` exponent controls how aggressively older posts decay out of the feed.
